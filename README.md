# lab_with_file
Постановка задачи: Дан файл с записанными разными символами, наша задача сжать файл следующим алгоритмом: берем 1 блок из 8 элементов, в блоке берем 1 элемент с двоичным кодом 01110101(например) и в следующие 7 элементов записываем в старший разряд соответствующий бит из 1 элемента (начиная с младшего разряда) таким образом получаем блок из 7 элементов , каждый элемент содержит соотвест бит в старшем разряде(0 или 1): 10010101.
    Чтобы разжать нужно взять блок из 7 элементов , создать буффер который будет содержать результирующий восстановленный блок из 8 элементов.в 1 элемент  достаем биты из старших разрядов , параллельно записывая элементы в буффер, а в конце записываем в финальный файл. 
Реализация работы происходит путем написания 2 функций : Compress DeCompress , которые реализовывают сжатие и разжатие.


Для удобства будем возвращать в функциях коды ошибок, если функции вернули равен 1, то все отработано штатно, иначе:
// 0 - ошибка файл пустой
// 2 - ошибка открытия файла(передан неправильный путь)
// 3 - ошибка fprintf
// 4 - ошибка когда файл закрылся успешно , но содержит недопустимые символы(кириллица или пустой символ)
// 5 - ошибка когда файл не закрылся , но удаление произошло успешно
// 6 - ошибка когда файл не закрылся и не произошло удачное удаление


Перейдем к тестам:
sixteen.txt  весит 16 байт (содержит 16 элементов) после сжатия размер полученного файла second.txt равен 14 байт (14 элементов) это происходит тк у нас взялось 2 блока по 8 байт и они сжались до 7 , 7 * 2 =14 все верно, при обратном разжатии исходный файл совпадает с файлом который мы запускали(откуда начали туда и  пришли), значит сжатие и разжатие отработало штатно.

fifteen.txt весит 15 байт (содержит 15 элементов) после сжатия размер полученного файла second.txt равен 14 байт (14 элементов) это происходит тк у нас взялся 1 целый блок из 8 байт и сжался до 7 + хвост, состоящий из 7 байт, тем самым 7 + 7 =14 все верно, разжатие происходит корректно и размер равен 15 (то есть исходному)
pystota.txt Весит 0 байт (содержит 0 элементов) функция возвращает код ошибки 0, который отвечает за пустоту файла, закрываем файл.

kirill.txt  весит 13 байт (содержит 13 элементов (6 кириллица и 1 пробел)) функция вернет код ошибки 4 тк мы встретили недопустимый символ.

file.txt весит 8 байт (содержит 8 нулевых элементов с кодом 00000000) функция вернет код 4 и отработает штатно

ascii.txt весит 127 байт(содержит всю таблицу аскии кроме 0 символа тк иначе будет ошибка) функция отрабатывает корректно происходит сжатие до 112 символов(112 байт) и разжатие приводит к исходному файлу.

bigtext.txt весит более 8 МБ (много символов) функция отрабатывает штатно и возвращает 7 МБ ( то есть сжалось корректно ) и разжатие происходит корректно
